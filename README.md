# ContribLens

Personal GitHub contribution analytics — exposing what your commits are actually made of.

Embed a live, beautiful, premium stats card in your GitHub profile README in one line:

```markdown
![ContribLens](https://your-deploy.com/svg/your-username)
```

## What it does

Given a GitHub username, ContribLens:

1. Fetches your **public contribution history** across **ALL** repositories you've contributed to (both owned and external) from the GitHub API.
2. Applies **contribution rules** — intelligently ignores generated files, lock files, vendor code, minified files, rename-only commits, and bot noise.
3. Calculates your **true personal language usage** from the files you actually authored, down to the line of code.
4. Returns the result as **JSON** or a **Stunning SVG card** (with a GitHub-style segmented language bar, custom activity icons, and dynamic sizing) ready to embed.

## Endpoints

| Endpoint | Description |
|---|---|
| `GET /analytics/:username` | Full analytics snapshot as JSON |
| `GET /svg/:username` | Beautiful SVG card for embedding |
| `GET /health` | Health check |

### Query parameters

Both endpoints accept:

| Param | Default | Description |
|---|---|---|
| `window` | `1` | Years of history to analyze (`1`–`5`, or `all`) |
| `theme` | `default-dark` | Card theme (`default-dark`, `default-light`, `high-contrast`) |

### Example

```
GET /svg/torvalds?window=1&theme=default-dark
GET /analytics/torvalds?window=all
```

## Self-host

### Requirements

- Node.js ≥ 22
- A GitHub personal access token with `read:user` and `public_repo` scopes

### With Docker

```bash
docker build -t ContribLens .
docker run -e GITHUB_TOKEN=ghp_your_token -p 3000:3000 ContribLens
```

Then visit `http://localhost:3000/svg/your-username`.

### Without Docker

```bash
cp .env.example .env
# Edit .env and set GITHUB_TOKEN

pnpm install
pnpm build
node apps/api/dist/index.js
```

### Environment variables

| Variable | Required | Description |
|---|---|---|
| `GITHUB_TOKEN` | ✅ | GitHub PAT with `read:user`, `public_repo` |
| `GITHUB_USER_AGENT` | ❌ | User-agent header (default: `ContribLens/1.0`) |
| `PORT` | ❌ | Port to listen on (default: `3000`) |

## How it works

```
GitHub API
    │
    ▼
ContributionCollector    ← fetches commits, PRs, file diffs across all repos
    │
    ▼
ContributionRulesEngine  ← excludes lock files, generated code, etc.
    │
    ▼
LanguageAnalyticsEngine  ← maps qualified file changes → exact language % (TypeScript, Python, JSON, etc.)
    │
    ▼
ConfidenceModel          ← assesses data completeness
    │
    ▼
AnalyticsSnapshot        ← immutable, versioned result
    │
    ├──▶  /analytics/:username  →  JSON
    └──▶  /svg/:username        →  SVG card
```

## Rules applied

Files are **excluded** from language attribution when they match any of these rules to keep your stats clean:

- **Lock files** — `package-lock.json`, `yarn.lock`, `Gemfile.lock`, `Cargo.lock`, etc.
- **Generated code** — files with `@generated`, `Code generated by`, `DO NOT EDIT` headers
- **Generated paths** — `__generated__/`, `*.pb.go`, `*.min.js`, migration files, etc.
- **Vendor code** — `vendor/`, `third_party/`, `node_modules/` (if somehow committed)
- **Minified files** — files with lines exceeding 500 characters
- **Binary files** — images, fonts, compiled artifacts
- **Rename-only commits** — no actual code change
- **Merge commits** — bot-generated merge noise
- **Formatting-only** — whitespace/indent-only diffs

*Note: Valid markup (like Markdown/HTML) and configuration files (like JSON/YAML) that you actively author ARE counted towards your language profile, giving you a true 100% picture of your footprint.*
